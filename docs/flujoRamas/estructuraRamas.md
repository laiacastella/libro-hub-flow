
# 🌿 Estructura de ramas

### `main`

* Entorno de **producción**.
* Asociada a Vercel Production.
* Solo debe contener código estable, validado y listo para cliente final.

### `staging`

* Entorno de **preproducción / QA**.
* Rama de validación final antes de pasar a producción.
* Aquí comprobamos funcionamiento global, experiencia de usuario y posibles mejoras.

### `develop`

* Rama principal de **desarrollo**.
* Punto de integración del trabajo diario del equipo.
* Desde aquí nacen las ramas de nuevas funcionalidades o cambios.
