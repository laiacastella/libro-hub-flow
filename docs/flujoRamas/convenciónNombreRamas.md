# 🌱 Convención de nombres para ramas

Las ramas nuevas se crean siempre desde `develop`.

Formato recomendado:

```text
tipo/donde/breve-nombre-descriptivo
```

Ejemplos:

```text
feat/solicitudes/anadir-fecha-solicitud
fix/login/error-redireccion
refactor/dashboard/limpiar-componentes
style/navbar/ajustar-responsive
```

## Tipos recomendados

* `feat` → nueva funcionalidad
* `fix` → corrección de errores
* `refactor` → mejora interna sin cambiar comportamiento
* `style` → cambios visuales
* `docs` → documentación
* `test` → pruebas

> Evitar tildes, espacios y caracteres especiales en nombres de ramas.
